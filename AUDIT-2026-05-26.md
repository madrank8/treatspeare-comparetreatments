# Audit & Gap Analysis — comparetreatments

**Date:** 2026-05-26
**Scope:** Strategy / architecture / compliance plan (`strategy/*.md`, `compliance/*.md`, `START-HERE.md`, `PROJECT-STATUS.md`) versus the actual `website/` codebase state.
**Method:** Read the four strategy docs end-to-end; surveyed every file under `website/app/`, `website/components/`, `website/lib/`, `website/types/`; ran a real off-mount `next build`; spot-checked compliance-critical components and data files.

---

## Executive summary

1. **The site is in genuinely good shape.** Production build is green: 46 static routes compile clean on Next 16.2.6 / React 19.1 / Tailwind v4, ESLint zero warnings, TypeScript strict pass. Compliance discipline on Rx copy is real — every compounded-medication mention is hedged correctly, brand-drug names use the ® mark, every affiliate link goes through `/go/[slug]` with the right rel tokens and a `noindex`/`no-store` redirect.
2. **The compliance plumbing is half-built.** The hard FTC/FDA/drug-copy work is done; the *operational* compliance work (CMP + Consent Mode v2, GA4/GTM, real legal-page values, healthcare-attorney review, LegitScript application) is not. None of it can be skipped before paid spend.
3. **The biggest single risk is the brand name.** `SITE.name = "CompareTreatments.com"` is the literal competitor. PROJECT-STATUS confirms the real domain is "not finalized." Shipping under this name is a trademark and Bing-ad-policy risk. Pre-launch swap is mandatory, and many downstream artifacts (legal pages, schema, disclosure copy, sitemap) all reference it — so the swap is cheap *now* and expensive *later*.
4. **Plan-vs-reality has three structural gaps:** no `/articles` system (planned for the organic moat), no drug sub-hubs (`/weight-loss/semaglutide` etc.), and no CMS (Sanity is planned, not connected). All three are needed for Pillar 4 of the 2X blueprint (own the organic ground the competitor abandoned).
5. **One medical reviewer (`dr-elena-marsh`) is bylined on all 9 categories.** That contradicts the E-E-A-T plan and is the cheapest single E-E-A-T fix in the audit.
6. **Stack is Next 16, not Next 15 as PROJECT-STATUS claims.** Build is clean, so this isn't actively broken — but `website/AGENTS.md` correctly warns that any new code must consult `node_modules/next/dist/docs/`, not training data, because Next 16 has API breaks.

## 3. Compliance + E-E-A-T spot check

**What's right:**

- `AffiliateButton`: outbound `rel="sponsored nofollow noopener noreferrer"`, `target="_blank"`, routes via `/go/[slug]`, includes `data-brand` + `data-placement` for tracking, fires `trackAffiliateClick`. Compliance-correct.
- `DisclosureBanner`: rendered above every offer chart on every category hub. Copy: "Advertising disclosure: CompareTreatments is reader-supported. When you click through to a provider listed below, we may earn a commission. This never influences our scores, rankings or editorial opinions — our methodology is applied identically to every provider. [Read our full disclosure]." This is FTC-grade — clear, conspicuous, "may earn a commission" wording, links to the policy page, methodology-honesty disclaimer.
- Brand-review pages emit a **category-aware affiliate disclaimer** beside the verdict CTA, with treatment-specific side-effect language (different copy for ED vs weight-loss). Better than industry norm.
- `/go/[slug]` route: `force-dynamic`, 302 redirect, `X-Robots-Tag: noindex, nofollow`, `Cache-Control: no-store, max-age=0`, fresh `crypto.randomUUID()` subid per click. Sealed correctly.
- **Rx copy discipline** in `reviews.ts` (1,768 lines, all 16 reviews): compounded medication is *explicitly* described as "not an FDA-approved finished product" and "not the same as Ozempic®/Wegovy®/Zepbound®" everywhere it appears. Brand drug names always carry ®. Outcomes are hedged ("not appropriate for everyone," "a licensed clinician determines suitability," "results vary"). Insurance is hedged ("never guaranteed"). This is the rare affiliate site that actually follows the FDA Rx-advertising rules.
- `robots.ts` allows the four named AI crawlers. The plan said "do NOT block GPTBot, ClaudeBot, PerplexityBot, Google-Extended." ✓
- `MedicalReviewerByline` is rendered on hubs (when reviewer is present).
- `Author` schema includes `hasCredential`, `alumniOf`, `memberOf`, `sameAs` — proper Person markup for E-E-A-T.

**What's wrong or thin:**

- **`SITE.name = "CompareTreatments.com"`** — the literal competitor name, used throughout the site, schema, disclosure copy, footer, robots `host`, sitemap origin, and OpenGraph. PROJECT-STATUS confirms the real domain is "not finalized." Trademark + Bing/Google trademark-policy risk if shipped this way.
- **One medical reviewer** (`dr-elena-marsh`) for all 9 categories. Plan called for credentialed reviewers across specialties (MD/DO/NP/PharmD/RD). At minimum a category like Blood Testing or Sexual Health needs a different specialty reviewer to be credible.
- **Legal-page placeholders unresolved** (per PROJECT-STATUS § Action items): `[COMPANY LEGAL NAME]`, `[BRAND NAME]`, `[SITE_DOMAIN]`, `[BUSINESS MAILING ADDRESS]`, `[STATE OF FORMATION]`, three email addresses, phone, `[EFFECTIVE DATE]`, `[MEDICAL REVIEWER NAME, CREDENTIALS]`, `[EDITORIAL TEAM DESCRIPTION]`, `[KNOWN LIMITATIONS]`. All must be filled before publish.
- **No CMP / Consent Mode v2.** State privacy (CCPA + 7 other state laws + WA "My Health My Data") and Google/Microsoft ad-policy both require it. Default `analytics_storage` and `ad_storage` must start `denied` until the user consents; non-essential tags must fire only after consent. None of this is in the codebase. Hard pre-launch blocker.
- **No GA4 / GTM** installed. `trackAffiliateClick` is a stub. Without measurement, the paid-channel cannot be operated (ROAS = unknown).
- **No DMCA-agent registration** signal in the codebase. The `/dmca` page exists but the U.S. Copyright Office designated-agent registration is an external task — flag it.
- **TrustBar headline stats:** `trustStats` is imported from `lib/data/site-content.ts`. The plan calls out "no fake social-proof counters" as a Do Not. The stats need to be either real or carefully framed editorial language. (I didn't read site-content.ts in this audit; verify before launch.)
- **Sitemap excludes legal/trust pages** — minor SEO/trust gap; legal-page indexation is a positive trust signal for YMYL sites.
- **`SearchAction` URL template** in `lib/schema.ts` `buildWebSiteSchema` references `/search?q={search_term_string}`, but no `/search` route exists. Rich Results validator will flag this as broken.

## 4. Strengths worth preserving

The codebase is doing several things genuinely well that should not be regressed:

1. **The brand-review template is rich** — breadcrumbs, review hero, two-paragraph hero summary, at-a-glance box, pros/cons, weighted rating breakdown, three-to-four H2 body sections, a real pricing table, verdict block with affiliate CTA *and* category-aware disclaimer, FAQ section, byline with author + reviewer + dates, related-brands grid, plus **four schemas** (Review, AggregateRating, MedicalWebPage, FAQPage). This is materially deeper than what comparetreatments.com publishes per landing page — exactly the "out-build the product" pillar in the plan.
2. **The category-hub two-column layout** with sticky sidebar (HowWeRankBox, SidebarOfferCard, DrugFactsBox where present, RecommendedModule) does the LP-vs-editorial-page balancing act well.
3. **`/go/[slug]` is correctly sealed** (force-dynamic + noindex + no-store + fresh subid). Many affiliate sites get one or two of these wrong.
4. **Rx-copy compliance discipline is real** — see §3 strengths list. Replicate it across the 7 missing categories.
5. **Schema builder library (`lib/schema.ts`)** is comprehensive and DRY — 10 builders covering Organization, WebSite, BreadcrumbList, ItemList, FAQPage, WebPage, MedicalWebPage, AggregateRating, Review, AuthorProfile, ReviewMedicalWebPage.
6. **Type system is the implicit Sanity migration spec** — when Sanity gets wired, the field-by-field mapping is already done.

## 6. Risks (ranked)

1. **Trademark / ad-policy on competitor name use.** The site is currently keyed to "CompareTreatments.com" verbatim. Bing's trademark policy disallows running ads that imply association with a third party; Google's is similar. A working title is fine internally, but the brand swap is a launch blocker.
2. **Unfilled legal-page placeholders + no attorney review.** Publishing the legal pack with `[COMPANY LEGAL NAME]` etc. visible is reputational and legal exposure. Plan calls for healthcare-advertising counsel review; that should be the gating step.
3. **CMP / Consent Mode gap is a state-privacy violation in WA, CA, CO, CT, VA, UT, TX, OR, MT** if any non-essential pixel fires before consent. Compliance roadmap explicitly demands this. Currently no pixels fire because none are installed, but the moment GA4 goes in without a CMP, this becomes live exposure.
4. **LegitScript is the critical-path gating item for paid spend.** $975 + $2,150/yr, takes weeks-to-months, gates Bing + Google health ads. PROJECT-STATUS lists it as a Niro action item; without a status update on it, the launch timeline is unknown.
5. **Single medical reviewer across all 9 categories** weakens the E-E-A-T signal that's the core differentiator vs the competitor.
6. **Next 16 breaking changes.** Build is clean now, but every new feature has to consult `node_modules/next/dist/docs/`. Training-data Next.js patterns may be wrong. (AGENTS.md is explicit about this.)
7. **Affiliate-link click logging is stubbed.** Without server-side logs you cannot reconcile partner-reported clicks with site-reported clicks, which is the standard fraud-and-shaving check in affiliate ops.
8. **No tests, no link checker.** Internal linking is a load-bearing SEO surface; a broken internal link in production today wouldn't be caught.
9. **FDA enforcement momentum** in GLP-1 (100+ warning letters Sept 2025, 30 more March 2026) raises the cost of *any* slip in compounded-drug copy. The repo's current discipline is good — protect it during the 7-category content fill-in.
10. **Domain not chosen.** Cascades into LegitScript application, schema canonical URLs, brand-name use, OG image generation, email addresses on legal pages, business-entity registration. Every day this drifts is a day the rest can't finalize.

## 8. Recommended next move

Three viable options, ranked by expected ROI on this audit.

**Option A — Compliance hardening sprint (recommended first).** Close P0-5 through P0-13 in code (a week of focused work, no external dependencies). In parallel, push P0-1 through P0-4 + P0-10 forward externally. **Why:** none of the content work matters until paid spend can run; paid spend can't run until LegitScript + CMP + GA4 + brand swap. P1 work shipped before P0 isn't deployable, so it's frozen value.

**Option B — Articles + drug-sub-hubs sprint (organic moat first).** P1-1 → P1-4 first (~1–2 weeks). **Why:** the plan's whole thesis is "take the abandoned organic ground." The articles + drug-sub-hubs are the only artifacts that build the organic moat that the competitor doesn't have. Risk: this content is published either under the wrong brand (forced rewrite after rebrand) or behind the wall of "not deployable yet."

**Option C — Brand-review fill on the 7 thin categories.** P1-5 (~1–2 weeks). **Why:** every hub currently links to a ranking with mostly `hasReview: false` entries. Filling those reviews replicates the proven weight-loss/ED quality across all verticals. Same risk as Option B: brand-name leak.

**My read:** do **Option A first**. Specifically — get the brand decided this week, swap the codebase to it in one PR, fill the legal placeholders, install CMP + Consent Mode + GA4 in one combined PR, register DMCA, and ship to Vercel preview. That unblocks everything else. P1 work after that has a real surface to land on.

If the brand can't be locked this week, **Option C** is the safest holding pattern — brand-review content for the 7 thin categories has the smallest brand-name surface (the reviews say "TrimRx," "Bluepeak Men's Health," etc., not "CompareTreatments") and is straight value-add to a future paid-channel landing page.

**What I would not start with:** Option B (`/articles` + drug sub-hubs). High-effort, high-value, but the value is undeliverable until P0 ships, and the content is brand-leaky enough that a rebrand mid-stream forces a meaningful rewrite.
